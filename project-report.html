VIT ShuttleAI — Web Mini Project Report Chapter Abstract

Abstract

Campus transportation is a critical component of the daily academic experience at large universities such as Vellore Institute of Technology (VIT). With thousands of students, faculty, and staff commuting across a sprawling campus every day, the absence of a real-time, intelligent shuttle management system results in significant inefficiencies — including long waiting times, route uncertainty, and poor fleet utilization.

VIT ShuttleAI is a fully functional, browser-based web application designed to address these challenges through real-time shuttle tracking, intelligent route visualization, and a student-facing dashboard. The system provides live GPS-simulated bus positions on an interactive satellite map of the VIT Vellore campus, with all four operational shuttle routes (Alpha, Bravo, Charlie, and Delta) displayed as colour-coded polylines that faithfully follow the actual campus road network.

The application is built entirely on client-side web technologies — HTML5, CSS3, and Vanilla JavaScript — with the Leaflet.js library rendering a real-world Esri satellite map. The multi-page architecture includes a Home page with live preview map, a Fleet Dashboard with occupancy and performance metrics, a Schedules page with timetable data, an Alerts & Notifications page, and a Sign-In portal supporting student and administrator access flows.

Key design principles include a vibrant "Electric Campus" neon dark-mode design system, smooth 60-fps bus animation, driver identification cards visible to passengers, and an "Advanced Intuitive Navigation" radar scan effect on map initialization.

The project demonstrates comprehensive application of web technologies including responsive CSS Grid/Flexbox layouts, CSS custom properties (design tokens), Leaflet.js interactive maps with custom DIV icons, real-time DOM animation via requestAnimationFrame, modal-based authentication UI, and intersection observer-driven scroll animations.

Keywords: Real-time tracking, Leaflet.js, Campus navigation, Shuttle management, Satellite map, CSS animations, Multi-page web application, VIT Vellore.

VIT ShuttleAI — Web Mini Project Report Chapter 1 – Introduction

Chapter 1 — Introduction

1.1 Overview of the Project

VIT ShuttleAI is a real-time, AI-assisted web application designed to modernize the shuttle management experience at VIT Vellore. The platform enables students to view live shuttle locations on an interactive satellite map, check arrival schedules, receive service alerts, and access fleet occupancy data through an intuitive dashboard — all directly in a web browser without installing any application.

1.2 Problem Statement

Students at VIT Vellore currently have no reliable way to determine (a) where a campus shuttle is at any given moment, (b) when the next shuttle will arrive at their stop, or (c) how crowded the shuttle is likely to be. This forces students to wait at stops indefinitely or miss classes. The proposed system provides a digital solution to all three pain points.

1.3 Objectives

  • Display real-time (simulated) shuttle positions on an accurate satellite map of VIT Vellore campus.
  • Represent all four operational shuttle routes as road-following coloured polylines.
  • Show driver information (name, rating, contact) visible to passengers when they click a bus.
  • Provide a fleet dashboard with speed, occupancy, and on-time performance metrics.
  • Display route schedules and live arrival countdowns for each campus stop.
  • Implement a student and administrator sign-in portal with role-based access design.
  • Deliver a responsive, mobile-friendly design consistent across all pages.

1.4 Scope of the Project

The current version of VIT ShuttleAI is a front-end prototype demonstrating all major UI/UX features. Bus positions are simulated using path-interpolation animation (not connected to live GPS hardware in this version). The project covers five HTML pages, shared CSS/JS design system, interactive Leaflet maps, and a modular component architecture ready for backend integration.

Chapter 2 — Literature Survey

2.1 Existing Systems

2.1.1 Google Maps Transit Layer

Google Maps provides public transit overlays for major city bus networks. However, it does not support custom campus shuttle routes or provide driver-level information relevant to private institutional operations. Its API cost and data-privacy requirements make it unsuitable for building a fully custom campus solution.

2.1.2 Moovit Campus

Moovit offers campus mobility solutions used by some international universities. It requires backend GPS integration through proprietary hardware and does not support custom design systems or institution-specific branding at the interface level.

2.1.3 Transloc Rider (Now TransitTrack)

Transloc is commonly used by US university campuses for shuttle tracking. While feature-rich, it is a closed SaaS platform. No existing Indian campus transit application provides real-time tracking through an open, browser-based interface tailored to VIT's road layout.

2.2 Proposed System & Novelty

Unlike the above systems, VIT ShuttleAI is:

  • Open & customizable — built entirely with open-source web standards and free-tier map APIs.
  • Campus-specific — all route waypoints, stop names, and boundaries are hand-calibrated against the actual VIT Vellore satellite imagery.
  • Driver-transparent — passengers can see the assigned driver's name, photo, rating, and trip history by clicking any live bus marker.
  • Deployable without hardware — the system works as a progressive enhancement; GPS data can be injected via a WebSocket connection without changing the frontend architecture.

Chapter 3 — System Design

3.1 System Architecture

VIT ShuttleAI follows a multi-page static web architecture. Each page is an independent HTML document that loads shared global styles (style.css) and shared JavaScript logic (shared.js), augmented by page-specific CSS and JS files.

Architecture Overview

Browser → HTML Pages → Shared CSS/JS → Leaflet.js → Esri Satellite Tile API
All pages are served by a static file server (e.g., npx serve) or any web host.

3.2 Module Description

  • style.css — Global design system: CSS custom properties (tokens), navigation, modals, buttons, scroll effects.
  • shared.js — IST real-time clock, sign-in modal, scroll animations, parallax effects, counter animation.
  • pages/map.js — Leaflet map initialization, satellite tile, route polylines, animated bus markers with driver info, stop markers, route filter logic.
  • pages/home.js — Preview mini-map with satellite tile, campus boundary, route lines overlaid.
  • pages/dashboard.css / home.css / map.css — Page-specific styles for each module.

VIT ShuttleAI — Web Mini Project Report Chapter 5 – Implementation & Features

Chapter 5 — Implementation & Features

5.1 Home Page (index.html)

The home page serves as the entry point with a full-viewport hero section featuring animated neon orbs, a grid background, and a real-time IST clock in the navigation bar. A satellite preview map of VIT campus is embedded with all four route lines drawn using Leaflet.js. The hero section contains live stat counters (fleet size, daily ridership, average wait time) animated via IntersectionObserver. Feature cards link to all major sections of the platform.

5.2 Live Map Tracker (map.html)

The map page displays a full-screen Esri satellite map of VIT Vellore campus. Four shuttle buses animate in real-time along road-following polyline paths using requestAnimationFrame() at 60fps. Each bus marker shows the driver's name, avatar, star rating, and trip count in a popup card. A collapsible sidebar lists all buses (with driver cards) and all campus stops. A radar scan "Advanced Intuitive Navigation" animation plays on first load. A route filter panel allows isolating individual routes.

5.3 Fleet Dashboard (dashboard.html)

The dashboard presents a comprehensive operational view of the VIT fleet. Key metrics displayed include: total active buses, buses on-schedule, daily ridership count, and average wait time — each shown in animated stat cards. A performance table lists each bus with columns for speed, occupancy percentage (shown as a colour-coded progress bar), route name, ETA to next stop, and current driver. A "Campus Mood" card shows weather and footfall context.

5.4 Schedule & Alerts Module

Schedules Page (schedules.html)

Displays a tabular timetable for all four routes with departure times and frequency. Includes a live "next bus" countdown for the nearest upcoming departure for each route.

Alerts Page (alerts.html)

Lists service alerts categorized as Delays, Route Changes, and General Notices. Filter chips allow the user to view alerts by category. Each alert card shows severity level, affected route, and timestamp.

5.5 Authentication & Sign-In System

A modal-based sign-in portal is accessible from the navigation bar on all pages. The modal contains two tabs — Student Login (accepting @vit.ac.in email addresses) and Admin Login (accepting institutional admin credentials). Input validation provides real-time feedback. The sign-in UI is fully designed and implemented; backend authentication is ready for integration.

Chapter 7 — Conclusion & Future Work

7.1 Conclusion

The VIT ShuttleAI project successfully demonstrates the design and development of a comprehensive, real-time campus shuttle tracking and management web application using modern front-end technologies. The project applies core web development concepts — including semantic HTML5, advanced CSS3 animations and layout systems, vanilla JavaScript DOM manipulation, and the Leaflet.js mapping library — to solve a real-world campus transportation problem.

The system achieves all seven stated objectives: live animated bus tracking on a satellite campus map, road-following route visualization, driver transparency for passengers, a comprehensive fleet dashboard, a schedule and alerts module, and a fully designed authentication interface. The codebase is modular, well-organized, and structured for straightforward backend integration.

This project provided practical experience in responsive web design, API integration, performance optimization via requestAnimationFrame, design systems using CSS custom properties, and user experience design — all directly aligned with the Web Technologies curriculum.

7.2 Future Enhancements

  • Integrate a real-time GPS feed via WebSocket or MQTT for live bus positioning.
  • Connect @vit.ac.in Firebase Authentication for student and admin login.
  • Build a Node.js or Django REST API backend for dynamic schedule and alert data.
  • Add a Progressive Web App (PWA) manifest for mobile installation and offline support.
  • Implement AI-based ETA prediction using historical traffic and ridership data.
  • Add push notifications for delay alerts via the Web Push API.
